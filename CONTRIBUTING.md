# Contributing Guidelines  

## ❗️ Before you start...  

- Make sure an [issue](https://github.com/LabelWorkshop/ios/issues) for your feature/fix is open and you have gotten permission from a team member
    - If there is already an assigniee for the issue it's very likely taken
- Don't create large pull requests with refactors without a disscusion
- Make sure no one else has opened a PR for the same issue

## 🧑‍💻 Development
Open the project in the latest version of [Xcode](https://developer.apple.com/xcode/) or [Xcode-beta](https://developer.apple.com/download/applications/). For testing have an iOS Simulator of 17.1 or later (preferably the newest).

## 🎨 Code Style
- ~~No more than 100 characters per line~~ Actually don't worry about this for now
- Use [camelCase](https://en.wikipedia.org/wiki/Camel_case) for variables and functions
- Use [UpperCamelCase](https://en.wikipedia.org/wiki/Camel_case) for classes and structs

## 👨‍🎨 Commit & PR Style
- Use [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) as a guideline for PR titles
- Use the default provided PR template unless your PR doesn't change code
